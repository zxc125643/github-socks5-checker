# 部署说明

## 一、建立节点仓库

1. 在 GitHub 新建一个公开仓库。
2. 把本目录全部文件上传到仓库根目录。
3. 打开仓库 `Settings` -> `Actions` -> `General`。
4. 在 `Workflow permissions` 选择 `Read and write permissions`。
5. 打开 `Actions`，手动运行一次 `Update SOCKS5 list`。

GitHub Actions 任务成功后，每天北京时间 10:15 自动更新。当前这台 Mac 也会通过 `launchd` 在 00:25、06:25、12:25、18:25 自动运行并推送数据。

```text
data/socks5.json
data/socks5.txt
data/results.json
data/status.json
```

edgetunnel 使用的地址为：

```text
https://raw.githubusercontent.com/zxc125643/github-socks5-checker/main/data/socks5.json
```

## 二、更新 Cloudflare Worker

1. 备份当前 Worker 代码和变量。
2. 用 `edgetunnel/_worker.js` 替换当前 Worker 代码。
3. 保留原有的管理员密码、UUID、KEY、KV 等变量和绑定。
4. 新增文本变量：

```text
变量名：SOCKS5_LIST_URL
变量值：https://raw.githubusercontent.com/zxc125643/github-socks5-checker/main/data/socks5.json
```

5. 部署 Worker。
6. 打开你的域名 `/admin` 并登录。
7. 在反代设置中选择 SOCKS5，点击原有的“获取更多 SOCKS5”按钮。

根路径出现 `Welcome to nginx!` 是 edgetunnel 的默认伪装页面，不代表部署失败。

## 三、筛选规则

- 先调用 `https://check.socks5.cmliussss.net/check` 获取出口 IP、ASN、公司和风险标记。
- 再从运行脚本的机器本机，通过 SOCKS5 真实访问两个 HTTPS 目标：
  - `https://www.gstatic.com/generate_204`
  - `https://www.cloudflare.com/cdn-cgi/trace`
- 默认两个目标都通过才算真正可用，避免只靠第三方检测导致假阳性。
- 参考 `yxip_git_action` 的纯净度思路，对出口 IP 计算 0 到 5 分：bogon、Tor、proxy、VPN、abuser、高风险 ASN/公司都会扣分。
- `data/results.json` 保存所有本机确认可用的节点及纯净度字段。
- `data/socks5.json` 和 `data/socks5.txt` 优先发布干净节点；没有干净节点时，默认发布本机可用但带 dirty/warning 标记的备用节点。
- `data/socks5.txt` 是带注释的人工查看格式：`socks5://地址:端口 #地区 /纯净度 /本机延迟`。机器对接建议继续使用 `data/socks5.json`。
- 当前严格模式下 `RETAIN_PREVIOUS_ON_EMPTY=false`，如果没有任何可发布节点，会发布空列表，不会继续上传旧的失效结果。

免费公开代理不可信，不要用于账号密码、支付、后台管理或任何敏感数据。

## 四、可调参数

在 `.github/workflows/update-socks5.yml` 中可以调整：

```text
MAX_CANDIDATES：每天检测的候选数量，默认 2000
MAX_RESULTS：最终发布数量，默认 100
CONCURRENCY：并发检测数，默认 24
TIMEOUT_MS：单节点超时，默认 35000 毫秒
MAX_LATENCY_MS：第三方检测最大延迟，默认 15000 毫秒
LOCAL_CHECK：是否开启本机真实连通性验证，默认 true
LOCAL_TIMEOUT_MS：本机访问单个目标超时，默认 10000 毫秒
LOCAL_REQUIRED_SUCCESSES：本机目标通过数量，默认 2
MIN_PURITY_SCORE：干净节点最低纯净度，默认 4.2
PUBLISH_ONLY_CLEAN：是否只发布干净节点，默认 false
RETAIN_PREVIOUS_ON_EMPTY：零结果时是否保留旧列表，当前默认 false
```

候选源位于 `config/sources.txt`，每行一个公开列表地址。

## 五、这台 Mac 的定时任务

本机仓库路径：

```text
/Users/macos/Documents/codex 2/github-socks5-checker
```

```text
scripts/macos_update_socks5.env
```

```bash
scripts/macos_update_socks5.sh
```

```bash
launchctl print gui/$(id -u)/com.zxc125643.github-socks5-checker
```

```bash
tail -f logs/macos_update_socks5.out.log
tail -f logs/macos_update_socks5.err.log
```
